Wordbook

Skill used: Java

  • The output is a group of HTML files.
  • There is a top-level index which lists each term in the wordbook, and separate pages for each of the terms with their definitions.
  • Clicking on a term in the index shall take you to the page for that term and its associated definition.
  • Moreover, clicking on any term in the wordbook that happens to appear in a definition shall take you to the page for that term and its associated definition.
  • Every term consists of a single "word" (i.e., a term contains no whitespace characters).
  • The terms appear in the wordbook index in alphabetical order.
  • The entire wordbook is generated from an input file.
  • The input consist of a single term on the first line, its definition on the next, an empty line on the next, another term on the next line, its definition on the next, an empty line on the next, and so on.
  • The input shall continue in this manner through the definition of the last term, which shall end with its terminating empty line.
  • The program does not check for invalid input.
  • The program asks the user for the name of the input file and for the name of a folder where all the output files will be saved (the output folder already exist and is not created by the program).
  • The top-level index is named index.html and the page for each term is named term.html where term is the actual term.
  • The user input is the complete relative or absolute path as the name of the input file, and does not augment the given path in any way, e.g., it does not supply its own filename extension.
  • For example, a reasonable user response for the name of the input file could directly result in the String value "data/terms.txt".
